# Edifice Construction — сайт строительной компании

Многостраничный презентационный сайт строительной компании Edifice для заказчиков, партнёров и соискателей. Сайт рассказывает об услугах и технологиях компании, показывает реализованные объекты и даёт быстрый доступ к контактам. Публикуется как статический сайт на GitHub Pages.

## Технологии

- HTML5, CSS3, нативный JavaScript
- Яндекс Карты API — карта объектов на странице «Проекты»
- Google Fonts (Inter)
- Статические изображения и брендинг в `assets/`
- GitHub Pages — публикация из ветки `main`
- БД, серверный код и сборщик не используются

## Структура

```
.
├── Главная.dc.html          # главная страница компании
├── Услуги.dc.html           # услуги
├── Проекты.dc.html          # каталог объектов: фильтры, карта, модальное окно
├── Технологии.dc.html       # технологии и качество
├── support.js               # общие интерактивные сценарии
├── assets/
│   ├── brand/               # логотип и favicon
│   └── projects/            # фотографии объектов
└── docs/
    ├── PROJECT_STATUS.md    # актуальное состояние и точка продолжения
    └── DECISIONS.md         # решения и ограничения
```

## Архитектура

- Статический multi-page сайт: каждая страница работает автономно и подходит для GitHub Pages.
- Данные карточек проектов и сопоставление изображений находятся в `Проекты.dc.html`; это исключает зависимость от внешнего API и обеспечивает быстрый рендер.
- Фотографии хранятся локально в репозитории и подключаются относительными путями — сайт не зависит от сторонних хостингов изображений.
- Интерактивность реализована нативным JavaScript, без фреймворка: для небольшого статического сайта это упрощает поддержку и деплой.

## Соглашения

- Имена статических изображений: латиница, kebab-case, например `renovation-lihachevsky.jpg`.
- Пути к ресурсам — относительные (`./assets/...`).
- Новые страницы сохраняются в текущем формате `*.dc.html`, чтобы не сломать существующую навигацию.
- Текст интерфейса — на русском; `alt` у содержательных фотографий содержит название объекта.
- При изменении карточки проекта синхронно проверять данные, изображение в `projectImages`, карточку и модальное окно.
